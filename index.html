Comandos do Git

Git status - Para ver o que tem de pendentes

Git add . - Para add todos os arquivos

Git commit -a -m “mensagem” - Para poder commitar os arquivos

Git push - para enviar todos os arquivos commitados para o servidor ( git hub )



No caso do repositório ainda não existir no GitHub iremos seguir esse passo-a-passo abaixo:

1 - De um GIT INIT na pasta que vc deseja que receba comandos do git, se vc clonou repositório que já estava no gitHub ou gitLab não tem necessidade de fazer isso.

2 - Git status - para ver o status

3 - git add + nome do arquivos ou pasta que vc quer add


Ex: git add teste.txt

Obs: se vc der o comando “ git add .” Ele irá add tudo.

4 - git commit -m + nome que desejamos colocar

Essa flag -m : é um comando para colocação de mensagem. Ex: git comia - m “firas commit”

5 - git branch -M master - para criar uma branch master


6 - criar uma origem ao meu projeto, ou seja, para ele se comunicar com Git

Git remote add origin git@github.com:Gabriel-Newton-dev/Curso_git_1 Para se comunicar com o gitHub, sincronizar

7 - Git push -u origin master


8 - git remote -v : para ver se as origens estão corretas

QUANDO O PROJETO JÁ ESTÁ NO GIT E NO GITHUB TEMOS APENAS QUE ATUALIZAR AS INFORMAÇÕES OU ADD ALGUM ARQUIVO CRIADO
1 - Se criar uma outra pasta dentro do projeto.
- Git status - para ver o que tem de pendente para add ou commitar.
- Git add . - se quiser add tudo ou - git add + nome do arquivo que quer add
- Quando der Git status novamente vai ver que a fala dele terá mudado

obs: no vsCode

. podemos ver quando criamos um novo arquivo ele fica com a flag U
. Quando fazemos alguma alteração em algum arquivo que já esta commitado no projeto - fala M
. Quando rodamos o comando git add - fica com a flag A, ou seja, pronta para Commitar
